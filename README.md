# Bootcamp @42SP linguagem C
Este repositório contém códigos desenvolvidos no basecamp C da 42saopaulo de Janeiro/Fevereiro de 2022.

# Sobre a 42

42 é uma iniciativa global de educação que propõe uma nova forma de aprender tecnologia: sem professores,
sem salas de aula, alunos aprendendo com seus colegas (aprendizagem entre pares), com um
metodologia que desenvolve tanto a computação quanto as habilidades para a vida. A 42 é grátis para quem estiver
aprovado em seu processo seletivo. O chamado "Basecamp", um bootcamp de programação C de 22 dias intensos,
é a última etapa do processo seletivo para se tornar um "cadete" (aluno da 42 SP).

# Skills desenvolvidas durante o basecamp
# Hard-skills

* # Linguagem C
	>A linguagem primária aprendida no basecamp é C: lidar com variáveis, ponteiros
	alocação de memória, macros, além de criar funções e programas,
	incluindo o uso e criação de bibliotecas e Makefiles.

* # Shell
	>Os dois primeiros dias do basecamp são dedicados ao aprendizado do básico
	ao intermediário comandos shell para navegar com facilidade pelo terminal.

* # Git
	>Todos os projetos são enviados por meio de um repositório git remoto,
	que exige que os alunos fiquem á vontade com tal sistema.
  
# Soft-skills 
  
 * # Autoaprendizagem e proatividade
   >Na 42, a autoaprendizagem está no centro da sua metodologia de aprendizagem entre pares. 
   Lá não tem professores. O processo de aprendizagem é baseado em pesquisar no Google, Youtube,
   perguntando aos nossos colegas e conhecimento empírico, cada aluno aprendendo em seu próprio
   ritmo.
   
 * # Trabalho em equipe
   >Nos finais de semana, é proposto um projeto de equipe de 48h chamado "Rush": um grupo de 3 membros 
   aleatoriamente escolhido pelo sistema da escola deve desenvolver, de forma colaborativa, um 
   projeto totalmente em execução. O grande desafio aqui é igualar os diferentes níveis de conhecimento
   e encontrar uma metodologia de trabalho comum que permite que todos os membros da equipe
   contribuam para o projeto.
   
 * # Gerenciamento de tempo e resiliência
   >Temos 22 dias para conseguirmos cumprir os desafios e projetos, porém nós mesmo somos os gerenciadores
   do nosso tempo, então exige o desenvolvimento dessas habilidadaes para não "aforgar-se" em meio à
   enorme carga de trabalho exigida pelo programa ininterrupto.
